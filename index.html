Intro

Version control (VC) has been a mainstay of software development for 40 years, but recent advances in VC frameworks, namely the distributed model popularized by Git, offer opportunities to address many of the challenges faced by big data analysis. In addition to the traditional software development necessities of versioning and collaborative development, data analysis presents some unique demands of VC software. These are inherent to the analysis model, in which input data and parameters are processed to produce reports.

  1. Spawn multiple related analyses (child analyses) from a single template
  2. Update those children and generate reports automatically as the parent analysis is refined
  3. Bind or "marry" reports with the analysis software, input data, and parameters that were used to produce them
  4. Version large files
  5. Compare reports

This white paper will explore possible solutions offered by Git and related software for handling big data analysis, as well as identify some areas where further development is needed.
